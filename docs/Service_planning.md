# 방청로그 (Bangcheong-Log) 서비스 상세 기획서

## 1. 서비스 개요 (Service Overview)

### 1.1 서비스명
**방청로그 (Bangcheong-Log)**

### 1.2 의미
'방청'과 '기록(Log)'의 합성어. 내가 참여한 방청 경험이 단순한 관람을 넘어 하나의 소중한 기록(Vlog, Diary)이 된다는 의미를 담음.

### 1.3 슬로건 (Slogans)
*   **Main:** "나의 첫 번째 방청 기록, 방청로그"
*   **Sub 1:** "본방보다 빠른 현장 기록, 방청로그"
*   **Sub 2:** "당신의 일상에 방송을 더하다, 방청로그"
*   **Catchphrase:** "세상에서 가장 빠른 본방사수"

### 1.4 타겟 오디언스 (Target Audience)
*   **Primary:** 방송 현장의 생생함을 즐기고 기록하기 좋아하는 Z세대 및 밀레니얼 세대 (Vlog 감성 보유자).
*   **Secondary:** 좋아하는 그룸/연예인의 스케줄을 따라다니며 응원하는 팬덤(Fandom).
*   **Tertiary:** 이색적인 데이트나 친구와의 추억을 만들고 싶어 하는 일반 대중.

### 1.5 디자인 컨셉 (Design Concept)
*   **Icon:** 슬레이트(Slate) 또는 녹화 버튼(● REC)을 모던하고 심플하게 재해석.
*   **Tone & Manner:** 감성적이고 트렌디한 무드. (Vlog 썸네일 스타일의 UI, 파스텔 톤 또는 깔끔한 다크 모드)

## 2. 핵심 기능 (Key Features)

### 2.1 방청 정보 통합 제공 (Aggregation)
*   **기능:** 주요 방송사(KBS, MBC, SBS 등) 및 케이블 채널의 방청 신청 페이지를 크롤링하여 한곳에서 보여줌.
*   **세부 내용:**
    *   프로그램별 방청 신청 기간, 녹화 장소, 출연진 정보 제공.
    *   신청 마감 D-Day 카운트다운 표시.
    *   '음악 방송', '토크쇼', '공개 코미디' 등 카테고리별 필터링.

### 2.2 맞춤형 알림 서비스 (Notification)
*   **기능:** 사용자가 관심 등록한 프로그램이나 키워드에 맞춰 알림 발송.
*   **세부 내용:**
    *   **키워드 알림:** 특정 연예인(예: 아이유, 유재석) 출연 시 즉시 푸시 알림.
    *   **마감 임박 알림:** 신청 마감 1시간 전/하루 전 리마인드 알림.
    *   **당첨 발표 알림:** (가능 시) 당첨자 발표일 알림 서비스.

### 2.3 간편 응모 가이드 (Easy Apply Guide)
*   **기능:** 복잡한 방송사별 회원가입 및 응모 절차를 쉽게 안내.
*   **세부 내용:**
    *   각 방송사별 응모 페이지 바로가기 링크 (Deep Link).
    *   응모 팁(사연 잘 쓰는 법, 필수 지참물 등) 제공하는 '방청 꿀팁' 섹션.

### 2.4 나만의 방청 기록 (My Log)
*   **기능:** 신청 내역 관리 및 다녀온 후기 기록.
*   **세부 내용:**
    *   **신청 캘린더:** 내가 응모한 스케줄을 달력 형태로 확인.
    *   **방청 다이어리:** 사진, 짧은 영상, 감상평을 남길 수 있는 피드(Feed) 기능.
    *   **배지 시스템:** '첫 방청 성공', '음방 마스터' 등 활동에 따른 디지털 배지 및 티켓 발급.

## 3. 사용자 흐름 (User Flow)

1.  **탐색 (Discover):** 앱 실행 -> 최신/인기 방청 목록 확인 (D-Day순 정렬).
2.  **관심 (Interest):** 특정 프로그램 상세 정보 확인 -> '알림 받기' 또는 '관심 등록'.
3.  **응모 (Action):** '응모하러 가기' 버튼 클릭 -> 해당 방송사 페이지로 이동하여 신청 수행.
4.  **기록 (Log):** (응모 후) 캘린더에 자동/수동 등록 -> (관람 후) 사진과 함께 후기('로그') 작성.
5.  **공유 (Share):** 작성한 로그를 SNS(인스타그램, 트위터 등)에 공유 가능한 이미지 형태로 내보내기.

## 4. 시스템 요구사항 (System Requirements)

### 4.1 클라이언트 (App/Web)
*   **플랫폼:** 모바일 웹 (Mobile Web) 우선 개발 후 앱(React Native/Flutter) 확장 고려.
*   **UI/UX:** 모바일 퍼스트, 직관적인 카드형 UI.

### 4.2 서버 (Backend)
*   **크롤러 (Crawler):** 주기적(Daily/Hourly)으로 방송사 사이트 스크래핑 (Python/Node.js).
*   **데이터베이스 (DB):** 프로그램 정보, 사용자 정보, 알림 설정, 로그 데이터 저장.
*   **알림 서버:** 푸시 알림(FCM) 또는 카카오톡 알림톡 연동.

---

## 5. 기대 효과 (Expected Outcome)
*   **사용자:** 번거로운 방청 정보 검색 시간을 단축하고, 놓치기 쉬운 기회를 잡을 수 있음.
*   **문화:** 방청 경험을 '기록'으로 남기는 새로운 문화를 형성하여 방송 콘텐츠에 대한 몰입도 증대.
