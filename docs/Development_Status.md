# 방청로그 (Bangcheong-Log) 개발 현황 및 로드맵

## 1. 개발 완료 (Developed) ✅

### 1.1 프로젝트 환경 설정
*   **프레임워크**: Next.js 14 (App Router) + TypeScript
*   **스타일링**: Tailwind CSS v4 + Framer Motion (애니메이션)
*   **PWA**: 모바일 앱처럼 설치 가능한 웹 앱 (Manifest 적용)
*   **테마**: 10-20대 타겟의 "Deep Dark" & "Neon Gradient" 테마 (다크모드 기반)

### 1.2 UI/UX 컴포넌트
*   **Mobile-First 레이아웃**: 데스크톱에서도 앱처럼 보이는 `MobileWrapper` 적용.
*   **내비게이션**: 인터랙티브한 하단 `BottomNav` (홈, 응모, 로그, MY).
*   **공통 컴포넌트**:
    *   `Card`: 글래스모피즘(유리 효과)이 적용된 방송 카드.
    *   `Button`: 호버 시 스케일이 커지는 젤리 같은 버튼.
    *   `Badge`: 프로그램 카테고리 태그.

### 1.3 페이지 (Front-End Only)
*   **홈 (Home)**:
    *   더미 데이터를 활용한 '마감 임박' 방청 목록 노출.
    *   캐치프레이즈 히어로 섹션 애니메이션.
*   **응모 (Apply) & 로그 (Log)**:
    *   기본 레이아웃 및 준비 중(Placeholder) 페이지 구현.

## 2. 개발 예정 (To Be Developed) 🚀

**핵심 가치: 정보 통합(Aggregation) 및 알림(Notification)**

### 2.1 백엔드 및 데이터베이스
*   **데이터베이스 구축**:
    *   `Program`: 방송명, 녹화일, 응모 기간, 출연진 정보 저장.
    *   `User`: 사용자 정보 및 관심 키워드 저장.
*   **API 개발**:
    *   프론트엔드와 통신할 REST API 또는 Server Actions 구현.

### 2.2 크롤러 (Crawler) - **[핵심]**
*   **방송사 연동**: KBS, MBC, SBS 등 주요 방송사 방청 신청 페이지 자동 수집.
*   **스케줄러**: 6시간마다 주기적으로 데이터를 갱신하여 최신성 유지.
*   **자동 분류**: 수집된 데이터를 카테고리(음악, 토크, 코미디)별로 자동 태깅.

### 2.3 알림 시스템 (Notification) - **[핵심]**
*   **키워드 알림**: 사용자가 등록한 연예인/방송 키워드 매칭 시 알림 발송.
*   **마감 임박 알림**: 신청 마감 하루 전/1시간 전 리마인드.
*   **채널**: 브라우저 Push API 또는 이메일/카카오톡 연동 고려.

### 2.4 사용자 기능
*   **로그인/회원가입**: 소셜 로그인 (카카오, 구글) 연동.
*   **방청 로그 작성**: 다녀온 방청의 후기(텍스트+사진) 남기기.
*   **딥링크 연결**: '응모하기' 버튼 클릭 시 해당 방송사 신청 페이지로 바로 이동.
